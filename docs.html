Documentación de la API

1. Registro de Usuario

URL: /api/register/

Método: POST

Autenticación: No requerida

Body (JSON):

{
    "username": "usuario123",
    "email": "usuario@example.com",
    "password": "contraseña123",
    "cellphone": "1234567890"
}

2. Inicio de Sesión

URL: /api/login/

Método: POST

Autenticación: No requerida

Body (JSON):

{
    "username": "usuario123",
    "password": "contraseña123"
}

3. Crear Ship

URL: /api/ship/create/

Método: POST

Autenticación: Requiere Token

Headers:

Authorization: Token {tu_token}

Body (JSON):

{
    "name": "Pedido1",
    "lat": 10.123456,
    "long": -75.123456
}

4. Listar Ships

URL: /api/ships/

Método: GET

Autenticación: Requiere Token

Headers:

Authorization: Token {tu_token}

5. Actualizar Ship

URL: /api/ship/{id}/

Método: PUT

Autenticación: Requiere Token (Solo el creador puede modificarlo)

Headers:

Authorization: Token {tu_token}

Body (JSON):

{
    "name": "Pedido Actualizado"
}

6. Eliminar Ship

URL: /api/ship/{id}/

Método: DELETE

Autenticación: Requiere Token (Solo el creador puede eliminarlo)

Headers:

Authorization: Token {tu_token}

7. Crear Cedi

URL: /api/cedi/create/

Método: POST

Autenticación: Requiere Token (Solo Admins)

Headers:

Authorization: Token {tu_token}

Body (JSON):

{
    "name": "Cedi 1",
    "lat": 10.123456,
    "long": -75.123456
}

8. Listar Cedis

URL: /api/cedis/

Método: GET

Autenticación: No requerida

9. Obtener Métricas de Entrega

URL: /api/cedi/metrics/

Método: GET

Autenticación: No requerida
